# 音频采样工具设计文档

## 项目概述
基于 UGUI 开发的音频采样与音符标记工具,用于音乐节奏游戏的谱面制作。

---

## 核心需求

### 1. 精确对音需求
- 在音频播放过程中,能够精确定位到音乐的具体时间点
- 根据 BPM(每分钟节拍数)和节拍细分,将时间转换为音符索引
- 支持点击或按键标记当前播放位置的音符
- 为谱面制作提供可视化的音频波形参考

### 2. 播放控制需求
- 支持音频、视频、Timeline 的同步播放
- 支持播放、暂停、停止、快进、快退操作
- 支持拖拽进度条跳转到指定位置
- 实时显示播放进度信息(时间、节拍、音符索引)
- 支持拖拽时预览音频(可以听见升/降调的音乐声，即使没有在播放，只要拖拽就能听见声音)

### 3. 可视化需求
- 以节拍为单位显示音频波形
- 动态加载可视范围内的节拍单元
- 高亮显示当前播放位置的音符
- 区分已选中和未选中的音符状态

---

## 功能模块

### 一、播放器交互模块 (`BasicPlayerInteraction`)

#### 1.1 多媒体播放控制
**功能描述:**
- 统一管理 AudioSource、VideoPlayer、PlayableDirector 三种播放器
- 提供统一的播放状态控制接口

**核心功能:**
- **播放状态管理:** 暂停(0)、播放(1)、停止(2)
- **进度控制:** 获取和设置播放位置,支持跨媒体类型的进度同步
- **时长获取:** 自动根据播放类型返回正确的总时长

#### 1.2 音乐节拍计算
**功能描述:**
- 根据 BPM 和节拍细分长度,将播放时间转换为音乐节拍信息

**核心功能:**
- **Bit(节拍)计算:** `PlayProgress * BPM / 60`
- **Note(音符)索引计算:** `PlayProgress * BPM * BitLength / 60`
- **SPN(每音符秒数):** `60 / (BPM * BitLength)`

#### 1.3 UI 交互
**功能描述:**
- 进度条拖拽时精确跳转到目标位置
- 播放/暂停按钮状态切换
- 实时更新播放信息显示

**核心功能:**
- **拖拽检测:** 通过 EventTrigger 监听 Slider 拖拽事件
- **信息格式化:** 支持自定义格式字符串显示播放数据(如 `{ProgressByTime}`, `{Bit}`, `{Node}`)
- **图标切换:** 根据播放状态自动切换播放/暂停图标

---

### 二、节拍管理模块 (`BitManager`)

#### 2.1 动态节拍池管理
**功能描述:**
- 根据播放位置动态加载和卸载可视范围内的节拍单元
- 优化性能,避免一次性加载全部节拍

**核心功能:**
- **可视节拍数量计算:** `Mathf.CeilToInt(Menu.rect.width / (NotePerBit * NoteWidth))`
- **前向加载:** 当播放位置接近最后一个节拍时,自动加载下一个节拍
- **后向加载:** 当播放位置回退接近当前加载范围内的第一个节拍时,在前面插入新节拍
- **自动销毁:** 超出可视范围的节拍单元会被销毁释放

#### 2.2 音频波形加载
**功能描述:**
- 从 AudioClip 中提取完整的音频采样数据
- 为波形可视化提供数据源

**核心功能:**
- **采样数据提取:** `clip.GetData(audioSamples, 0)`
- **格式验证:** 检查 AudioClip 加载类型必须为 `DecompressOnLoad`
- **多声道处理:** `totalSamples = clip.samples * clip.channels`

#### 2.3 音符选择管理
**功能描述:**
- 记录用户标记的音符位置
- 支持添加/移除选中状态

**核心功能:**
- **当前音符追踪:** 自动追踪播放位置对应的音符单元
- **选中列表维护:** 管理所有已选中的音符列表
- **空格键快捷标记:** 按空格键快速标记当前播放位置的音符

---

### 三、节拍单元模块 (`BitUnit`)

#### 3.1 节拍容器管理
**功能描述:**
- 作为一个节拍的容器,管理该节拍下的所有音符单元
- 根据节拍细分长度(BitLength)创建对应数量的音符

**核心功能:**
- **音符生成:** 根据 `BitLength` 参数创建子音符单元
- **索引显示:** 显示当前节拍的索引编号
- **位置同步:** 根据播放进度实时更新在时间轴上的位置

#### 3.2 位置计算与渲染
**功能描述:**
- 根据播放进度动态计算节拍单元在UI上的位置

**核心公式:**
- **水平位置:** `(BitIndex - BitProgress) * BitLength * NoteWidth`
- **尺寸:** `BitLength * NoteWidth`

---

### 四、音符单元模块 (`NoteUnit`)

#### 4.1 音符状态可视化
**功能描述:**
- 根据音符的不同状态显示不同颜色
- 提供清晰的视觉反馈

**状态定义:**
- **普通状态:** 未选中且非当前播放位置
- **播放状态:** 当前播放位置的音符(高亮)
- **选中状态:** 用户已标记的音符
- **选中播放状态:** 当前播放到已标记的音符

#### 4.2 音频波形可视化
**功能描述:**
- 在每个音符内部显示该时间段的音频波形
- 帮助用户精确判断音符的位置

**核心功能:**
- **采样范围计算:**
  - 起始采样点: `661500 * Index / BPM`
  - 结束采样点: `661500 * (Index + 1) / BPM`
- **分段采样:** 将一个音符的采样数据分成 `SampleCountPerNote` 段显示
- **振幅计算:** 对每段采样取绝对值求和并归一化
- **波形条绘制:** 使用子对象 RectTransform 绘制垂直波形条

#### 4.3 交互功能
**功能描述:**
- 支持点击或快捷键切换音符的选中状态
- 播放到选中音符时触发音效提示(预留接口)

**核心功能:**
- **选择切换:** `ToggleSelection()` 添加或移除选中状态
- **自动高亮:** 播放到当前音符时自动更新视觉状态
- **音效触发:** 预留了 TapSound 播放接口

## 技术特点

### 1. 动态加载优化
- 仅加载可视范围 ±2 个节拍单元
- 根据播放方向(前进/后退)智能加载
- 自动回收超出范围的对象

### 2. 精确的时间-音符转换
- 基于 BPM 和节拍细分的数学计算
- 支持浮点数精度的节拍进度(`BitDouble`)
- 整数音符索引用于精确标记

### 3. 多层级架构
- **Manager 层:** 全局管理和数据控制
- **Bit 层:** 节拍容器,组织音符
- **Note 层:** 最小单元,负责可视化和交互

### 4. 波形可视化
- 实时从 AudioClip 提取采样数据
- 分段显示提高细节精度
- 可调节缩放系数(`SampleZoom`)

---

## 应用场景

1. **音乐游戏谱面编辑器**
   - 制作者可以边听音乐边标记音符
   - 波形可视化辅助判断节奏点

2. **音频同步调试**
   - 精确对齐游戏事件与音乐节拍
   - 验证音频播放与游戏逻辑的同步性

3. **节奏分析工具**
   - 分析歌曲的节拍结构
   - 导出标记数据用于其他系统
